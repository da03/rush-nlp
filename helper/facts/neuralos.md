# NeuralOS

## What it is
- NeuralOS is a neural generative model that simulates an entire operating system: every screen pixel is generated by a neural network, with no real kernel, no installed applications, and no fixed menus or icons - just weights.
- It is Yuntian's vision for the most flexible user interface: a fully generative UI, which he calls the final form of chatbots.
- Demo: https://neural-os.com ; code: https://github.com/yuntian-group/neural-os ; paper: arXiv:2507.08800 (ICLR 2026).
- Authors: Luke Rivard, Sun Sun, Hongyu Guo, Wenhu Chen, and Yuntian Deng (University of Waterloo and the National Research Council Canada); supported by the Laude Institute (Slingshots program).

## Using the demo
- To interact, move your mouse inside the interaction box (the screen area) and the model generates the next frame in response to where your cursor goes.
- Click (left-click or right-click) to perform click actions, and use your keyboard to type - just like a real desktop.
- Controls on the page let you set the number of sampling (diffusion) steps - more steps look better but are slower - and toggle options such as "Use RNN" and "Auto Input".
- Because every frame is generated by a neural network (a diffusion model) in real time, the demo can be slow and the picture may look soft or imperfect - that is expected; there is no real screenshot underneath, only generated pixels.

## How it works
- Two parts: an RNN tracks the operating-system state - like a kernel, with constant memory at any sequence length - and a diffusion renderer generates each screen frame, like a desktop manager.
- It is autoregressive: given the past frames and the user's input (mouse movement, click, or keypress), it predicts the next frame, then feeds that frame back as input - responding to every user action, indefinitely, rather than playing a fixed clip.
- Cursor position was a key challenge (under 0.1% of pixels but driving every click); encoding the cursor's (x, y) as a Gaussian spatial map (an extra input channel) cut cursor error from about 130 pixels to about 1.6 pixels.

## Data and training
- Trained on about 2,000 AI-agent demonstrations (from a Claude 3.5 Sonnet computer-use agent) plus about 120,000 random-exploration demonstrations; the random data prevents spurious correlations (e.g. learning that hovering near a button closes it).
- Data was collected on 64 Docker containers running Ubuntu XFCE at 512x384 resolution and 15 fps (about 12 TB).
- Training is a multi-stage pipeline: RNN pretraining, then joint RNN + diffusion training, then scheduled sampling (feeding the model its own outputs to stop error drift), then context extension (32 to 64 frames, generalizing to 256).

## What it can do
- It can even learn to launch and play Doom from fabricated/synthetic demonstrations - Doom was never actually installed; the training data was synthesized.
- Takeaway: anything you can demonstrate, even synthetically, can become a usable interface - opening the door to UIs that can't be hand-coded in fixed formats like HTML.
- It is like a driving simulator for computer-use agents: agents can learn interfaces from demonstrations without risking a real OS, accounts, or money.
- Resolution has grown quickly: 256x256 (2024), 512x384 (2025), 1024x768 (2026).
