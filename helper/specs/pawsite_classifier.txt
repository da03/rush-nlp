Classify a message from a visitor on the ProgramAsWeights website (programasweights.com). Decide whether the visitor is best served by sending them to a specific link, or whether it is a question that needs a written answer.

Return ONLY one of the labels below, nothing else.

Labels and when to use each:
{{LINKS}}
- question: any request for INFORMATION that should be answered in words - what PAW is, how it works, how to install it (the command), how to write specs, compiler/runtime differences, limits and rate limits, the license, whether it is free, privacy of data, examples, errors, or whether PAW fits a task.

Rules:
- "Where is / show me / take me to / open / browse X" is usually a link. "What is / how does / can I / is it / what license" is a question.
- Getting-started, learning, guides, quickstarts, and the API reference -> docs (a page to read).
- How to install / the install command (pip, npm) -> question (we answer the command inline). But the source code, repository, or package SOURCE -> code.
- The license ("what license is this under"), whether it is free, and whether it is OPEN SOURCE ("is this open source") -> question (we answer it: yes, MIT). Use terms only for "terms of service" / legal terms. (The source code repository itself is still code.)
- Source code, GitHub, repositories, an SDK, or a package (the Python SDK, the JavaScript/npm SDK, "the python sdk", "npm package") -> code (a fuzzy selector then picks Python vs JavaScript vs the org page). Exception: "how do I install" or "what should I install" is a question (we answer the command).
- API keys, sign-in, login, account -> settings. The visitor's OWN compiled programs -> profile.
- MAKING a new program (make / create / build / compile a classifier / function / model / program, from scratch) -> playground (the compiler). FINDING an EXISTING program by what it does ("find a sentiment analyzer", "is there a program that does X") -> hub (the community gallery).
- The alien language game demo -> alien (even "play the alien game"); the avatar demo -> avatar. These are specific demos, NOT agents and NOT the about page.
- Bug reports, problems, suggestions, praise ("great tool"), or wanting to reach the team/email -> feedback.
- Greetings, thanks, small talk, and off-topic or nonsensical messages ("hi", "hello", "good morning", "thanks", "tell me a joke") are question - NEVER feedback and NEVER a link.
- When unsure, prefer question.
